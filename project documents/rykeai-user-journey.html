Safety protocol

Crisis Detection Flow

Every inbound message is screened before the AI coaching reply is generated. If a crisis signal is found, coaching is suppressed and the safety protocol activates immediately.

⚠️ Crisis Detection Protocol

Triggered by keyword match OR Claude confidence ≥ 0.65. Fail-safe: API error → assumes crisis=true.

01
Detection
Keyword fast-path (instant) OR Claude ML classifier. Returns crisis, confidence, dimension.
02
Hold Session
crisis_hold=true on user. Session set to CRISIS_HOLD. AI coaching suspended immediately.
03
Holding SMS
One of 3 empathetic holding messages sent instantly. Includes 988 Crisis Line reference.
04
Coach Alert
SMS + email to coach within 5 min. Contains user name, phone, alert ID. SLA monitored.
SAMPLE HOLDING MESSAGE
"Thank you for trusting me with this. What you're feeling is real and it matters. I'm connecting you with someone who can support you — please hang tight. If this is urgent, text 988 (Crisis Line) now."
RESOLUTION — COACH ACTION
Coach calls POST /v1/safety/alerts/{alertId}/resolve → user.crisis_hold reset → session reactivated → user receives: "Your coach has checked in. I'm here whenever you're ready — no rush. 💙"
